# Как запустить новый проект по этому шаблону

Это переносимый каркас для работы с Claude Code по схеме **оркестратор + сабагенты**
(главная сессия проектирует, дешёвые модели haiku/sonnet пишут код).
Кладёшь в новую папку — и работаешь так же, как в КонтентЗаводе.

## Что внутри

```
.claude/
  settings.json          ← включает Stop-хук (следит за свежестью STATE.md)
  hooks/check-state.ps1  ← сам хук (пути НЕ захардкожены — копируется как есть)
  agents/
    coder-simple.md      ← haiku: boilerplate, шаблоны, мелочь
    coder.md             ← sonnet: бизнес-логика, интеграции, тесты
    architect.md         ← opus: архитектура, сложный дебаг
CLAUDE.md                ← протокол работы (верх не трогать) + про проект (заполнить)
STATE.md                 ← состояние работы (точка входа в проект)
PLAN.md                  ← план по этапам с чекбоксами
TZ.md                    ← (создай сам) сырое ТЗ / идея проекта
КАК ПОЛЬЗОВАТЬСЯ.md       ← этот файл (в новый проект можно не копировать)
```

## Запуск нового проекта — по шагам

1. **Скопируй** в папку нового проекта: `.claude/`, `CLAUDE.md`, `STATE.md`, `PLAN.md`.
   (`КАК ПОЛЬЗОВАТЬСЯ.md` копировать не обязательно.)
2. **Положи своё ТЗ/идею** в `TZ.md` в той же папке — хоть одним абзацем, хоть как наберётся.
3. **Заполни в `CLAUDE.md`** только нижние секции (помечены `<…>`): «О проекте»,
   «Стек и ограничения», «Команды». Верхнюю часть («Протокол работы») НЕ трогай.
   Если лень — можешь не заполнять: попроси меня в первой же сессии, я заполню по TZ.md.
4. **Открой Claude Code именно в папке нового проекта** (важно — см. ниже про память и хук).
5. Скажи мне: «прочитай TZ.md, давай уточним ТЗ» — и пойдём по фазам: вопросы → стек → план → код.

`STATE.md` и `PLAN.md` дальше веду я (оркестратор) — руками их обычно не трогаешь.

## Важные нюансы

- **Открывай Claude Code в корне нужного проекта.** Память и Stop-хук привязаны к рабочей
  папке. Если запустить из чужой папки — подхватятся чужие настройки, а хук будет ругаться
  не на тот проект (ложные срабатывания).
- **Пути править не нужно.** Хук берёт корень проекта автоматически (`$CLAUDE_PROJECT_DIR`).
  Копируй `.claude/` как есть в любой проект.
- **Память у каждого проекта своя** и копипастом не переносится — заведётся заново.
  Общие предпочтения (русский, экономия токенов) можно один раз вынести в глобальную
  память Claude — попроси меня, если захочешь.
- **Коммиты — за тобой.** Сабагентам по правилам запрещены git-операции; я тоже без твоей
  команды не коммичу и не пушу.
- **Stop-хук — это фича, а не баг.** Он не даёт мне закончить ответ, если я изменил код,
  но забыл обновить `STATE.md`. Так состояние проекта всегда актуально.

## Как со мной работать эффективно (то, что зашло в КонтентЗаводе)

- Сложные/дорогие решения принимаю я, рутину делают дешёвые сабагенты — экономит токены.
- Не прыгаем в код из сырой идеи: сперва уточняем ТЗ (я задаю вопросы пачкой),
  потом архитектура/стек, потом план с этапами, потом реализация по порядку.
- Если что-то непонятно — я спрашиваю, а не угадываю.
- Каждый этап заканчивается обновлением STATE.md/PLAN.md — ничего не теряется между сессиями.
